# Note-Taking Structure

Write daily notes. Push to repository every day.

## Categories

- **routine**: review notes, planning for day and week
- **python**: code snippets, things learned, errors encountered
- **cs**: wiki pages, summarize important concepts
- **fun**: side projects, interesting things found
- **todo**: ideas, learning material to check later
- **lang**: English vocabulary, grammar notes
- **peoples**: who's important in IT and why
- **homework**: assigned reading, exercises to complete

## Example Daily Note

```markdown
# 2024-03-15

## routine
- Reviewed yesterday's notes on Python lists
- Plan: finish chapter 3 of Python Basics, do XO exercise

## python
- Learned list comprehensions: `[x*2 for x in range(10)]`
- Error: `IndexError` when list is empty — always check `len()` first

## cs
- Read about Big O notation: O(n) vs O(log n)
- Summary: binary search is O(log n) because it halves the search space each step

## todo
- Check out "Automate the Boring Stuff" chapter on web scraping
- Learn about Docker containers

## lang
- "it depends on" — not "it depends to"
- "to be familiar with" — useful phrase for tech discussions
```

---

> **Discipline**: push notes every day. Even one line counts. Consistency beats intensity.
